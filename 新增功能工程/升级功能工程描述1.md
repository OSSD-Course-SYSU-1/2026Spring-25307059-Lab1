## 🎯 核心功能分析

### 1. 抽奖转盘系统

#### 1.1 Canvas绘图实现
**文件**: `DrawModel.ets`

**核心功能**:
- 使用Canvas 2D API绘制精美的转盘界面
- 多层次视觉效果：花瓣装饰、外圈、内圈、扇形区域
- 弧形文字绘制技术
- 图片资源动态加载

**绘图流程**:
```
draw() → drawFlower() → drawOutCircle() → drawInnerCircle() 
       → drawInnerArc() → drawArcText() → drawImage()
```

**技术亮点**:
- 采用比例计算适配不同屏幕尺寸
- Canvas坐标系变换实现旋转效果
- 弧形文字逐字符绘制算法

#### 1.2 抽奖概率系统
**文件**: `DrawModel.ets` (calculatePrizeIndex方法)

**奖品等级与概率**:
| 等级 | 奖品 | 基础概率 | 颜色标识 |
|------|------|----------|----------|
| SUPER (★★★) | 西瓜 | 0.1% | 金色 #FFD700 |
| HIGH (★★) | 蛋糕 | 0.6% (动态) | 蓝色 #2196F3 |
| MEDIUM (★) | 汉堡 | 6% | 绿色 #4CAF50 |
| COMMON (-) | 谢谢参与 | ~93.3% | 灰色 #999999 |

**概率计算算法**:
```typescript
// 使用0-10000的整数范围提高精度
const rand = Math.random() * 10000;

// 保底系统影响高级奖品概率
if (pityCounter >= 30) {
  highRate += (pityCounter - 30) * 100; // 软保底
}

// 硬保底：50抽必出高级奖品
if (pityCounter >= 50) {
  return getHighPrizeIndex();
}
```

### 2. 保底机制 (Pity System)

#### 2.1 双重保底设计
**文件**: `CommonConstants.ets`

**保底参数**:
- **硬保底阈值**: 50抽 (PITY_THRESHOLD)
- **软保底起始**: 30抽 (SOFT_PITY_START)
- **历史记录上限**: 100条 (MAX_HISTORY_RECORDS)

**机制说明**:
1. **软保底** (30-49抽): 每抽增加1%的高级奖品概率
2. **硬保底** (50抽): 必定获得高级奖品（蛋糕或西瓜）
3. **保底重置**: 获得高级或超级奖品后计数器归零

#### 2.2 保底进度显示
**实现位置**: `CanvasPage.ets` + `StatisticsDialog.ets`

- 主界面底部实时显示保底计数器
- 统计对话框中显示进度条
- 显示剩余抽数提示

### 3. 抽奖模式

#### 3.1 单抽模式
**方法**: `performSingleDraw()`

**流程**:
1. 禁用按钮防止重复点击
2. 调用概率计算获取奖品
3. 播放转盘旋转动画 (4秒)
4. 显示奖品结果对话框
5. 更新统计数据和历史记录

#### 3.2 十连抽模式
**方法**: `performTenDraw()`

**特点**:
- 连续执行10次单抽逻辑
- 独立的保底计数（每次抽取独立判定）
- 网格布局展示10个结果
- 统计各等级奖品数量

### 4. 数据统计系统

#### 4.1 统计数据结构
**文件**: `DrawModel.ets` (DrawStatistics类)

```typescript
class DrawStatistics {
  totalDraws: number;      // 总抽数
  singleDraws: number;     // 单抽次数
  tenDraws: number;        // 十连抽次数
  pityCounter: number;     // 当前保底计数
  commonCount: number;     // 普通奖品数量
  mediumCount: number;     // 中等奖品数量
  highCount: number;       // 高级奖品数量
  superCount: number;      // 超级奖品数量
}
```

#### 4.2 统计展示
**文件**: `StatisticsDialog.ets`

**展示内容**:
- 保底进度可视化（进度条）
- 抽卡方式统计（单抽/十连）
- 奖品分布统计（各等级数量）
- 剩余保底抽数提示

### 5. 历史记录系统

**文件**: `HistoryDialog.ets`

**功能特性**:
- 记录最近100条抽奖记录
- 显示奖品图标、名称、稀有度
- 时间戳格式化显示 (月/日 时:分)
- 列表项交替背景色增强可读性

## 🔧 技术架构分析

### 1. 状态管理

**使用的装饰器**:
- `@State`: 组件内部状态管理
- `@Link`: 双向数据绑定
- `@CustomDialog`: 自定义对话框

**状态变量** (CanvasPage):
```typescript
@State drawModel: DrawModel          // 抽奖模型
@State rotateDegree: number          // 旋转角度
@State enableFlag: boolean           // 按钮使能标志
@State prizeData: PrizeData          // 单抽结果
@State tenDrawResults: PrizeData[]   // 十连抽结果
@State history: PrizeData[]          // 历史记录
@State statistics: DrawStatistics    // 统计数据
@State pityCounter: number           // 保底计数器
```

**aboutToAppear()**:
- 获取窗口实例
- 设置全屏显示
- 计算屏幕尺寸（px转vp）

### 3. Canvas使用

**初始化**:
```typescript
private settings: RenderingContextSettings = new RenderingContextSettings(true);
private canvasContext: CanvasRenderingContext2D = new CanvasRenderingContext2D(this.settings);
```

**绑定**:
```typescript
Canvas(this.canvasContext)
  .onReady(() => {
    this.drawModel.draw(this.canvasContext, this.screenWidth, this.screenHeight);
  })
```

✅ **用户体验**:

### 2. 可改进点

⚠️ **数据持久化**:
- 历史记录和统计数据未持久化
- 应用重启后数据丢失
- 建议: 使用Preferences或RDB存储

⚠️ **配置灵活性**:
- 概率参数硬编码
- 保底阈值固定
- 建议: 提取为配置文件

⚠️ **测试覆盖**:
- 缺少单元测试
- 缺少UI测试
- 建议: 添加测试用例

⚠️ **国际化完善**:
- 部分硬编码字符串
- 建议: 全部使用资源引用

⚠️ **无障碍支持**:
- 缺少无障碍标签
- 建议: 添加accessibility属性

### 3. 潜在风险

🔴 **精度问题**:
- 概率计算使用浮点数
- 可能存在精度误差
- 建议: 使用整数运算

🔴 **内存泄漏**:
- 图片资源未显式释放
- 建议: 在aboutToDisappear中清理

🔴 **并发安全**:
- 统计数据更新无锁保护
- 建议: 使用原子操作或锁机制
