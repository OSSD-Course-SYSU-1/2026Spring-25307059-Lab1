# CanvasComponent 抽奖转盘应用工程描述

## 项目概述

**CanvasComponent** 是一个基于HarmonyOS Canvas组件实现的抽奖转盘应用。该应用展示了如何使用CanvasRenderingContext2D进行自定义图形绘制，结合动画效果和自定义弹窗，实现一个完整的抽奖转盘功能。

## 项目基本信息

- **项目名称**: CanvasComponent
- **包名**: com.example.canvascomponent
- **版本**: 1.0.0
- **开发环境**: 
  - HarmonyOS SDK: 6.0.2 Release及以上
  - DevEco Studio: 6.0.2 Release及以上
  - HarmonyOS系统: 5.0.5 Release及以上
- **支持设备**: 直板机（标准系统）

## 核心功能模块

### 1. 主页面 (CanvasPage.ets)
- **功能**: 抽奖转盘主界面
- **组件**: 
  - `@Entry` 页面入口组件
  - `@Component` UI组件
- **核心特性**:
  - 使用Canvas绘制转盘界面
  - 响应式布局适配不同屏幕
  - 点击中心按钮开始抽奖动画
  - 使用animateTo实现旋转动画
  - 旋转5圈+随机角度制造悬念效果

### 2. 绘制模型 (DrawModel.ets)
- **功能**: 负责Canvas绘制逻辑
- **绘制内容**:
  1. 外层花瓣装饰
  2. 外层圆环和小圆点
  3. 内层圆环
  4. 内部扇形抽奖区域
  5. 扇形区域内的文字
  6. 扇形区域内的奖品图片
- **绘制方法**:
  - `draw()`: 主绘制方法
  - `drawFlower()`: 绘制花瓣
  - `drawOutCircle()`: 绘制外圆环
  - `drawInnerCircle()`: 绘制内圆环
  - `drawInnerArc()`: 绘制扇形区域
  - `drawArcText()`: 绘制弧形文字
  - `drawImage()`: 绘制奖品图片
  - `showPrizeData()`: 根据角度获取奖品数据

### 3. 数据模型
- **PrizeData**: 奖品数据模型
  - `message`: 奖品信息
  - `imageSrc`: 奖品图片路径
- **FillArcData**: 圆弧数据模型
  - `x`, `y`: 圆心坐标
  - `radius`: 半径
  - `startAngle`, `endAngle`: 起始和结束角度

### 4. 弹窗组件 (PrizeDialog.ets)
- **功能**: 显示中奖结果的弹窗
- **特性**:
  - 自定义弹窗样式
  - 显示中奖图片和信息
  - 点击确认按钮关闭弹窗

### 5. 常量定义
- **CommonConstants**: 通用常量
  - 角度常量: CIRCLE(360°), HALF_CIRCLE(180°)
  - 奖品数量: COUNT(6)
  - 动画时长: DURATION(4000ms)
  - 图片路径和尺寸
  - 绘制比例系数
- **ColorConstants**: 颜色常量
  - 花瓣颜色: FLOWER_OUT_COLOR, FLOWER_INNER_COLOR
  - 圆环颜色: OUT_CIRCLE_COLOR, INNER_CIRCLE_COLOR
  - 扇形颜色: ARC_PINK_COLOR, ARC_YELLOW_COLOR, ARC_GREEN_COLOR
- **StyleConstants**: 样式常量
  - 尺寸百分比: FULL_PERCENT(100%)
  - 字体粗细: FONT_WEIGHT(600)

## 技术特性

### 1. Canvas绘制技术
- **CanvasRenderingContext2D**: 使用2D渲染上下文进行绘制
- **坐标变换**: 使用translate和rotate进行坐标变换
- **路径绘制**: 使用beginPath、arc、fill等API绘制图形
- **文字绘制**: 使用fillText绘制弧形文字

### 2. 动画实现
- **显式动画**: 使用`animateTo`实现转盘旋转动画
- **缓动曲线**: 使用Curve.Ease实现先快后慢的效果
- **旋转控制**: 旋转5圈(1800°) + 随机目标角度

### 3. 响应式设计
- **屏幕适配**: 获取窗口尺寸并转换为vp单位
- **比例绘制**: 所有尺寸基于屏幕宽度比例计算
- **全屏布局**: 设置窗口全屏显示

### 4. 组件通信
- **状态管理**: 使用`@State`管理组件状态
- **数据绑定**: 使用`@Link`进行双向数据绑定
- **自定义弹窗**: 使用`CustomDialogController`管理弹窗

#### 功能描述

   - 获取窗口实例并设置全屏显示
   - 获取屏幕尺寸并转换为vp单位

2. **UI构建阶段** (`build`):
   - 使用Stack布局叠加Canvas层和按钮层
   - 配置Canvas的旋转属性
   - 设置背景图片和按钮点击事件

3. **用户交互流程**:
   ```
   用户点击中心按钮
   ↓
   禁用按钮防止重复点击
   ↓
   调用startAnimator()启动动画
   ↓
   生成随机角度确定中奖结果
   ↓
   执行转盘旋转动画(5圈+目标角度)
   ↓
   动画完成后显示中奖弹窗
   ↓
   用户点击确认按钮关闭弹窗
   ↓
   重新启用中心按钮
   ```

#### 前端代码分析
```typescript
// 核心状态管理
@State drawModel: DrawModel = new DrawModel();      // 绘制模型
@State screenWidth: number = 0;                     // 屏幕宽度
@State screenHeight: number = 0;                    // 屏幕高度
@State rotateDegree: number = 0;                    // 旋转角度
@State enableFlag: boolean = true;                  // 按钮启用状态
@State prizeData: PrizeData = new PrizeData();      // 中奖数据

// Canvas配置
private settings: RenderingContextSettings = new RenderingContextSettings(true);
private canvasContext: CanvasRenderingContext2D = new CanvasRenderingContext2D(this.settings);

// 动画启动方法
startAnimator() {
  let randomAngle = Math.round(Math.random() * CommonConstants.CIRCLE);
  this.prizeData = this.drawModel.showPrizeData(randomAngle);
  
  this.getUIContext().animateTo({
    duration: CommonConstants.DURATION,
    curve: Curve.Ease,
    onFinish: () => {
      this.rotateDegree = CommonConstants.ANGLE - randomAngle;
      this.dialogController.open();
    }
  }, () => {
    this.rotateDegree = CommonConstants.CIRCLE * CommonConstants.FIVE +
                        CommonConstants.ANGLE - randomAngle;
  });
}
```

1. **页面加载**: `aboutToAppear()` → 获取屏幕尺寸 → 初始化Canvas
2. **Canvas准备**: `onReady()` → `drawModel.draw()` → 绘制转盘
3. **用户交互**: 点击按钮 → `startAnimator()` → 动画执行
4. **动画完成**: `onFinish`回调 → 显示弹窗
5. **弹窗交互**: 点击确认 → 关闭弹窗 → 重置按钮状态

### 2. 绘制模型模块 (DrawModel.ets)

#### 功能描述
绘制模型模块负责Canvas的所有绘制逻辑，包括：
- 绘制转盘的各个视觉层
- 计算扇形区域和文字位置
- 管理奖品数据和图片绘制

#### 绘制流程
```
draw() 主绘制方法
├── translate() 坐标变换到中心
├── drawFlower() 绘制外层花瓣
├── drawOutCircle() 绘制外圆环和小圆点
├── drawInnerCircle() 绘制内层圆环
├── drawInnerArc() 绘制内部扇形区域
├── drawArcText() 绘制扇形内的弧形文字
├── drawImage() 绘制奖品图片
└── translate() 坐标还原
```

#### 关键技术点
1. **坐标系统**: 使用translate将原点移动到屏幕中心
2. **旋转绘制**: 使用save/restore保存和恢复绘图状态
3. **弧形文字**: 使用三角函数计算每个字符的位置和角度
4. **比例计算**: 所有尺寸基于屏幕宽度的比例系数

#### 扇形区域绘制算法
```typescript
drawInnerArc() {
  let colors = [/* 颜色数组 */];
  let radius = this.screenWidth * CommonConstants.INNER_ARC_RATIOS;
  
  for (let i = 0; i < CommonConstants.COUNT; i++) {
    this.fillArc(new FillArcData(0, 0, radius, 
      this.startAngle * Math.PI / CommonConstants.HALF_CIRCLE,
      (this.startAngle + this.avgAngle) * Math.PI / CommonConstants.HALF_CIRCLE), 
      colors[i]);
    this.startAngle += this.avgAngle;
  }
}
```

#### 弧形文字绘制算法
```typescript
drawCircularText(textString: string, startAngle: number, endAngle: number) {
  // 计算每个字符占用的弧度
  let angleDecrement = (startAngle - endAngle) / (textString.length - 1);
  let angle = startAngle;
  
  for (let i = 0; i < textString.length; i++) {
    // 计算字符位置
    let x = Math.cos(angle) * radius;
    let y = -Math.sin(angle) * radius;
    
    // 变换坐标和旋转角度
    this.canvasContext?.translate(x, y);
    this.canvasContext?.rotate(Math.PI / 2 - angle);
    this.canvasContext?.fillText(textString.charAt(i), 0, 0);
    
    angle -= angleDecrement;
    this.canvasContext?.restore();
  }
}
```

```typescript
showPrizeData(randomAngle: number): PrizeData {
  // 根据角度确定奖品区域(6个扇形，每个60度)
  for (let i = 1; i <= CommonConstants.COUNT; i++) {
    if (randomAngle <= i * this.avgAngle) {
      return this.getPrizeData(i);
    }
  }
  return new PrizeData();
}

getPrizeData(scopeNum: number): PrizeData {
  let prizeData: PrizeData = new PrizeData();
  switch (scopeNum) {
    case 1: // 西瓜区域
      prizeData.message = $r('app.string.prize_text_watermelon');
      prizeData.imageSrc = CommonConstants.WATERMELON_IMAGE_URL;
      break;
    case 3: // 笑脸区域(未中奖)
      prizeData.message = $r('app.string.prize_text_smile');
      prizeData.imageSrc = CommonConstants.SMILE_IMAGE_URL;
      break;
    case 4: // 蛋糕区域
      prizeData.message = $r('app.string.prize_text_cake');
      prizeData.imageSrc = CommonConstants.CAKE_IMAGE_URL;
      break;
    case 2:
    case 5: // 汉堡区域
      prizeData.message = $r('app.string.prize_text_hamburger');
      prizeData.imageSrc = CommonConstants.HAMBURG_IMAGE_URL;
      break;
    case 6: // 笑脸区域(未中奖)
      prizeData.message = $r('app.string.prize_text_smile');
      prizeData.imageSrc = CommonConstants.SMILE_IMAGE_URL;
      break;
  }
  return prizeData;
}
```

### 4. 弹窗模块 (PrizeDialog.ets)

```typescript
@CustomDialog
export default struct PrizeDialog {
  @Link prizeData: PrizeData;    // 双向绑定奖品数据
  @Link enableFlag: boolean;     // 双向绑定按钮状态
  
  build() {
    Column() {
      Image(this.prizeData.imageSrc)    // 中奖图片
        .rotate({ angle: CommonConstants.TRANSFORM_ANGLE })  // 旋转-120度
      
      Text(this.prizeData.message)      // 中奖信息

          this.controller?.close();      // 关闭弹窗
          this.enableFlag = !this.enableFlag;  // 恢复按钮状态
        })
    }
    .backgroundColor($r('app.color.dialog_background'))
  }
}
```

4. **状态同步**: 恢复主页面按钮的启用状态

### 10. 性能优化策略

#### 绘制优化
1. **Canvas状态管理**: 使用save()/restore()保存和恢复绘图状态
2. **路径重用**: 复用相同的绘制路径减少计算
3. **比例计算**: 基于屏幕宽度的比例系数，适配不同分辨率

#### 动画优化
1. **硬件加速**: 使用animateTo利用GPU硬件加速
2. **缓动曲线**: Curve.Ease提供自然的缓动效果
3. **状态管理**: 动画期间禁用交互防止冲突

#### 内存优化
1. **资源复用**: CanvasContext和ImageBitmap复用
2. **及时清理**: 动画完成后清理临时状态
3. **资源释放**: 弹窗关闭时释放相关资源
